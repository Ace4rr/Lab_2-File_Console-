# Лабораторная работа

## Описание

Данный проект представляет собой Docker-container, реализованный в рамках **Лабораторной работы №2**
Приложение-CLI-утилита (minishell-консоль,реализующая файловые методы), упакованная в Docker контейнер.
Внутри контейнера поддерживается интерактивный режим
### Запуск контейнера

* **Сборка Docker образа**: `docker build -t username/myapp:latest`.
* **Запуск контейнера**: `docker run -it username/myapp:latest`
* **Запуск контейнера с bind mount**:
Windows:`docker run -it -v ${PWD}\src:/app/src `username`/myapp:latest`
Mac OS/Linux:`docker run -it -v $(pwd)/src:/app/src `username`/myapp:latest`
* **Пуш в Докерхаб**:
`docker push ace4r/cli_file_manager:latest`
`docker push ace4r/cli_file_manager:1.0`

### Формат ввода
- Вводится команда оболочки (ls, cd, cat, cp, mv, rm и тп).
- Опции команд (-l, -r и тп) задаются через пробел.
- Поддерживаются как относительные, так и абсолютные пути.
- История команд логируется.

### Алгоритм работы
Пользователь вводит команду в minishell>.
Введённая строка парсится через argparse.
Вызывается соответствующий обработчик команды из модуля commands/.
Обработчик выполняет действие.
Все команды и ошибки логируются в shell.log.
Результаты выводятся на экран.
